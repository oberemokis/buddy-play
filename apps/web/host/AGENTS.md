# @sync/host

Shell на Module Federation. Потребляет ремоуты, владеет Pinia.

- **Порт**: 5173
- **Роль**: хост — нет `exposes`, только `remotes`

## Реестр ремоутов (`vite.config.ts`)

Карта `remotes` НЕ пишется руками — она выводится из манифеста ремоутов в `@sync/config` через `toRemotes()`. URL каждого ремоута можно переопределить переменной окружения `VITE_REMOTE_<NAME>_URL`. Пример ниже иллюстрирует структуру результата, но сам объект генерируется из манифеста:

```ts
remotes: {
  posts: {
    type: "module",
    name: "posts",
    entry: "http://localhost:5001/remoteEntry.js",
  },
}
```

## Типы ремоутов (`src/remotes.d.ts`)

Ambient-декларации модулей для каждого экспорта ремоута. Указывают на исходный компонент ремоута (только типы, стираются при сборке).

Этот файл АВТОГЕНЕРИРУЕТСЯ скриптом `scripts/gen-remotes.ts` из манифеста ремоутов в `@sync/config` — руками НЕ редактировать. Запускается командой `pnpm gen:remotes`, а также на хуках predev/prebuild.

Сейчас содержит:
- `posts/PostList` → корректно

## Дерево компонентов

- Локальных компонентов нет — только `App.vue`.
- `App.vue` асинхронно загружает компоненты ремоутов: `defineAsyncComponent(() => import("posts/PostList"))`.
- Оборачивайте в `<Suspense>` для запасного значения на время загрузки.
- Pinia создаётся здесь (`createPinia()` в `main.ts`) — это тот синглтон, который MF разделяет со всеми ремоутами.
